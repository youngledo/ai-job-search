# 中文面试回答模板

## STAR 结构

适合回答行为问题、项目经历和冲突处理。

```text
S - Situation 背景：
当时的业务/项目背景是什么？问题为什么重要？

T - Task 任务：
你负责什么？目标是什么？约束是什么？

A - Action 行动：
你具体做了什么？如何分析、沟通、推进和取舍？

R - Result 结果：
结果是什么？最好包含数据、交付物、上线效果、业务影响或复盘结论。
```

## CAR 结构

适合回答技术项目、复杂问题和业务推进。

```text
C - Context 背景：
问题出现在哪里？涉及哪些角色和限制？

A - Action 行动：
你采取了什么方案？为什么这么选？遇到阻力如何处理？

R - Result 结果：
最终效果是什么？有什么可证明产出？你学到了什么？
```

## 回答检查表

- 是否明确说清你的角色，而不是只说团队成果？
- 是否有事实、指标、交付物或外部反馈？
- 是否能解释关键取舍？
- 是否避免夸大职责、规模和结果？
- 是否能连接到当前岗位要求？

## 通用回答骨架

```text
这个问题我可以用【项目/经历】举例。

当时背景是【背景】，核心问题是【问题】。我负责【你的角色】，目标是【目标】。

我的做法分三步：第一，【行动 1】；第二，【行动 2】；第三，【行动 3】。

最终结果是【结果/指标/交付物】。这段经历对当前岗位的相关性在于【对应岗位要求】。

如果重新做，我会在【复盘点】上优化。
```

## 不确定问题的回答方式

```text
这个方向我没有完整的一线经验，但我有相邻经历：【相邻证据】。如果在这个岗位中遇到类似问题，我会先【第一步】，再【第二步】，同时向【角色/资料】确认关键假设。
```
